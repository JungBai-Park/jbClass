# 소프트웨어 요구사항 명세서 (SRS)

※ 본 문서에서 표현되는 상대 경로는 모두 프로젝트 루트를 기준으로 한다.

---

## 1. 프로젝트 개요

1. ConBox

   - 타 프로젝트에 쉽게 이식 가능한 커스텀 윈도우 컨트롤 **ConBox**를 제작한다.

   - ConBox는 wt.exe처럼 텍스트 입력과 출력을 모두 처리하는 터미널 컨트롤이다.

   - MFC 기반 CWnd 파생 클래스로 구현한다.

   - 모든 문자열은 **UTF-8 기반**이며, 문자열 입력 API는 `const char*`를 사용한다.
     - (C++ 문자열 리터럴을 그대로 넘길 수 있도록 `char*`가 아닌 `const char*`로 한다.)

2. ConExe

   - 타 프로젝트에 쉽게 이식 가능한 터미널 기반 응용프로그램 실행기 헬퍼 클래스 **ConExe**를 제작한다.
   - ConExe는 터미널 기반 응용프로그램을 실행하여 stdin/stdout/stderr을 리다이렉트 한다.
   - 리다이렉트된 입출력은 ConExe를 생성할 때 지정된 ConBox의 인스턴스로 인터페이스 할 수 있도록 제어한다.

---

## 2. ConBox

### 2.1 창 생성 API

```cpp
void open(CWnd* parent, int x0, int y0, int x1, int y1);
```
- 리소스 에디터 없이 좌표 지정만으로 창을 생성한다.

#### 그리드 크기 조회 API

```cpp
void client_size_for_grid(int cols, int rows, int& w, int& h) const;
```

- 가로 `cols` 칸(영문 기준) × 세로 `rows` 줄을 모두 담는 데 필요한 **클라이언트 영역 픽셀 크기**를 `w`/`h` 로 돌려준다. 안쪽 여백(`set_margin`)도 포함한 값이다.
- 호스트가 "80×24 로 띄우기"처럼 그리드 기준으로 창 크기를 정할 때 쓴다.
- 한 칸 픽셀은 폰트/DPI 로 정해지므로 `open()` 이후(셀 크기 확정 후) 호출해야 정확하다.

#### 현재 그리드 크기 조회 API

```cpp
int grid_cols() const;   // 현재 화면 그리드의 가로 칸 수
int grid_rows() const;   // 현재 화면 그리드의 세로 줄 수
```

- 현재 클라이언트 크기/폰트로 정해진 화면 그리드의 칸/줄 수를 돌려준다. `open()` 이후 유효하다.
- 호스트가 자식 콘솔(ConExe 의 의사 콘솔) 크기를 ConBox 그리드에 맞춰 시작할 때 쓴다.

### 2.2 폰트 설정 API

```cpp
void set_efont(const char* 폰트이름, float 크기, const char* 추가설정);  // 영문 폰트
void set_kfont(const char* 폰트이름, float 크기, const char* 추가설정);  // 한글 폰트
```

- 크기 단위: **포인트(pt)**. 소수점 크기(예: 10.5, 12.1)도 지정할 수 있다.
  - 다만 폰트는 최종적으로 정수 픽셀 높이로 래스터화되므로, 소수 크기는 어느 정수 픽셀에 떨어질지를 더 세밀하게 고르는 의미이며 같은 DPI에서 일부 값은 같은 픽셀이 될 수 있다.
- 기본값
  - 영문: Consolas, 13pt, Bold
  - 한글: 맑은 고딕, Bold (크기는 0 = 영문 높이에 맞춤. 아래 "한글 높이를 영문에 맞추는 옵션" 참조)

#### 추가설정 문자열 형식

LOGFONT 속성을 간결한 문자열로 지정한다. 숫자가 알파벳 앞에 오면 해당 속성의 정도(값)를 의미한다.

| 기호 | 속성 | 값 필요 여부 | 예시 |
|------|------|-------------|------|
| `B`  | Bold | 아니오 | `"B"` |
| `I`  | Italic | 아니오 | `"I"` |
| `U`  | Underline | 아니오 | `"U"` |
| `S`  | Strikeout | 아니오 | `"S"` |
| `W`  | Weight | 예 (앞에 숫자) | `"700W"` |

- 조합 예시: `"BI"` (Bold+Italic), `"400W"` (Weight=400), `"BU"` (Bold+Underline)

#### 한글 높이를 영문에 맞추는 옵션 (set_kfont 의 크기로 지정)

별도 토글 API 없이 `set_kfont` 의 크기 인자로 맞춤(match) 모드를 정한다.

- **크기 ≤ 0**: 맞춤 모드가 켜진다. 한글을 영문과 **같은 픽셀 높이**로 맞춘다. 이때 크기 값 자체는 무시된다. (단순히 같은 pt 로는 한글이 더 커 보이므로 실제 글자 높이가 같아지도록 보정한다.)
  - 줄 높이를 영문 기준으로 잡아 박스 그리기 문자(┌─┼ 등) 표가 세로로 촘촘히 이어진다.
  - 한글 자간(2칸 폭 안의 빈 공간)이 넓어 보이는 문제를 줄이기 위해, 한글은 가로로 찌그러뜨리지 않고(자연 폭 유지) 영문 칸 폭을 그 한글 폭에 맞춰 약간 좁힌다(영문 장평). 그 결과 한글이 두 칸을 거의 채워 자간이 좁아진다.
- **크기 > 0**: 맞춤 모드가 꺼진다. 한글은 지정한 크기 그대로이고, 영문도 원본 폭이며, 줄 높이는 두 폰트 중 큰 쪽이 된다.
- 아무 설정도 하지 않은 기본 상태는 맞춤 켜짐이다(기본 한글 폰트의 크기가 0).

#### 장평 비율 설정 API

```cpp
void set_kfont_fill(float fill_ratio, float emin_ratio);   // 기본값 (0.92, 0.7)
```

- 맞춤 모드일 때 한 칸 폭(장평)을 정하는 두 비율이다. 맞춤 모드가 꺼져 있으면 효과가 없다.
- `fill_ratio`: 한글 한 글자가 두 칸을 채우는 비율(기본 0.92). 키우면 영문 칸이 좁아져 한글 자간이 줄고, 줄이면 자간이 넓어진다.
- `emin_ratio`: 영문을 자연 폭의 이 비율 밑으로는 좁히지 않는 가독성 하한(기본 0.7).
- 한 칸 폭의 구체적 결정 규칙은 설계명세서의 `calc_cell_size` 항을 참조한다.

### 2.3 문자 렌더링

- 내부적으로 화면을 고정 `cols × rows` **셀 그리드**로 두고(2.4 참조), 모든 문자는 그 칸 위치에 그린다(가변폭 렌더링 금지).
- 한글은 영문의 **2배 폭**으로 렌더링한다(두 칸 차지).
- 폰트 또는 창 크기 변경 시 가로/세로 표시 가능 문자 수(cols/rows)를 재계산한다.
- 가로폭 초과 시 **자동 줄바꿈** 처리한다.
- 스크롤백이 있을 때만(내용이 화면을 넘을 때만) **세로 스크롤바**를 표시한다.

#### 안쪽 여백 (set_margin)

```cpp
void set_margin(int top, int left = -1, int bottom = -1, int right = -1);
```

- 클라이언트 영역 안쪽 여백(픽셀)을 둔다. 여백 안쪽에만 글자가 그려진다.
- 기본값은 사방 **10px**.
- CSS 단축 스타일로 생략한 변은 다른 변을 따른다(순서대로 평가).
  - `left` 가 음수면 `top` 을 따른다.
  - `bottom` 이 음수면 `top` 을 따른다.
  - `right` 가 음수면 (위에서 결정된) `left` 를 따른다.
  - 예: `set_margin(5)` → 사방 5, `set_margin(5, 8)` → 상하 5 / 좌우 8.
- `open` 이후 호출하면 창 크기 변경과 동일하게 가로/세로 표시 가능 문자 수를 재계산한다.

#### 박스/블록 그리기 문자 직접 렌더 (set_builtin_glyphs)

```cpp
void set_builtin_glyphs(int level);
```

- 박스/블록 그리기 문자를 폰트 글리프 대신 ConBox 가 **도형으로 직접** 그리는 수준을 정한다.
- 폰트의 블록 글리프는 칸을 가로/세로로 꽉 채우지 않아(특히 세로) 인접 칸 사이에 틈이 생긴다. 칸을 직접 칠하면 틈 없이 이어지고 폰트에 의존하지 않는다. (Windows Terminal 의 "Built-in Glyphs" 와 같은 방식)
- 수준
  - `0`: 끔 (전부 폰트로 그림)
  - `1`: 블록 요소(U+2580~259F: 반칸/사분면/풀블록)만 직접 그림 **(기본값)**
  - `2`: 1 에 더해 박스선도 직접 그림 — 단선 직교/정션(`─│┌┐└┘├┤┬┴┼`), 둥근 모서리(`╭╮╰╯`, **직각으로 대체**), 대각선(`╱╲╳`), 순수 이중선(`═║╔╗╚╝╠╣╦╩╬`).
- 단/이중 혼합선·점선·굵은선·음영은 어느 수준에서도 폰트로 그린다.

### 2.4 내부 버퍼 (셀 그리드)

- 화면을 고정 `cols × rows` **셀 그리드**로 관리한다. 셀 하나가 화면 한 칸이며 글자(UTF-16)와 전경/배경색을 가진다. 한글 등 2배 폭 글자는 lead 칸 + 빈 trail 칸 두 칸을 차지한다.
- 커서는 화면 내 **`(행, 칸)` 절대 위치**다(VT 시퀀스가 직접 이동시킨다).
- 화면 맨 위로 밀려난 줄은 **스크롤백**에 보관한다(상한을 넘으면 오래된 줄부터 버린다). 보기 위치는 스크롤백 + 화면 통합 인덱스로 관리한다.

### 2.5 출력 API

```cpp
void print(const char* text);
```

- 받은 UTF-8 바이트열을 **VT(ANSI 이스케이프) 시퀀스로 해석**해 화면 셀 그리드에 반영한다. ConPTY 자식(cmd/PowerShell/vim 등)의 콘솔 출력을 그대로 넘기는 용도다.
- 파서는 ConExe 가 출력을 청크로 나눠 여러 번 `print()` 로 넘겨도 시퀀스가 끊기지 않도록 **상태를 호출 간에 유지**한다.
- 해석하는 주요 동작:
  - 글자 출력 + 폭(`cols`) 초과 시 **자동 줄바꿈(autowrap)**. 커서가 화면 맨 아래에서 줄을 내리면 맨 윗줄을 스크롤백으로 밀어내며 스크롤한다.
  - C0 제어: `\r`(줄 맨 앞), `\n`(다음 줄), `\b`(백스페이스), `\t`(다음 **8의 배수** 탭 스톱), BEL 무시.
  - CSI: 커서 이동(CUU/CUD/CUF/CUB/CUP/HVP/CHA/VPA), 화면/줄 지우기(ED/EL), 색·속성(SGR), 커서 저장·복원, 커서 표시/숨김(DECTCEM `?25`).
  - SGR 색: 기본 16색 + 굵게(밝은색)·반전 + **256색·트루컬러**. OSC(창 제목 등)는 무시한다.
  - **풀스크린(TUI):** 대체 화면 버퍼(`?1049`/`?1047`/`?47`, `?1048` 커서 저장복원), 스크롤 영역(DECSTBM `CSI r`), 줄 삽입삭제(IL `CSI L`/DL `CSI M`), 문자 삽입삭제(ICH `CSI @`/DCH `CSI P`)·문자 지우기(ECH `CSI X`), 영역 스크롤(SU `CSI S`/SD `CSI T`). 대체 화면 동안은 스크롤백을 동결(보이지 않게)하고 복귀 시 주 화면·스크롤백을 되돌린다. 스크롤 영역이 설정되면 LF·RI·삽입삭제가 그 범위 안에서만 동작한다. vim/htop·claude.exe 같은 풀스크린 TUI 가 이 경로로 동작한다.
  - **자식 질의 응답:** 커서 위치(DSR `CSI 6 n` → `ESC[행;칸R`)·단말 상태(`CSI 5 n`)·디바이스 속성(DA `CSI c` → VT102 호환)을 입력 싱크(자식 stdin)로 응답한다.
  - **무시(오인 방지):** `<`/`=`/`>` 로 시작하는 프라이빗 시퀀스(2차 DA, kitty keyboard 프로토콜, XTMODKEYS 등)는 지원하지 않으며, 그 최종 바이트를 표준 명령(커서 복원 `u`·SGR `m` 등)으로 오인하지 않고 통째로 무시한다.
- 새 출력이 오면 스크롤이 **강제로 맨 아래로** 이동한다(대체 화면에서는 항상 화면이 보인다).

### 2.6 색상 API

```cpp
void set_color(COLORREF fg);         // 전경색(글자색) 설정
void set_bg_color(COLORREF bg);      // 배경색 설정
```

- 설정 이후 호출되는 `print()`에 적용된다. 단 자식 출력의 **SGR 색 시퀀스**가 이후 색을 덮어쓸 수 있다(SGR `0`/`39`/`49` 는 기본색으로 되돌린다).
- 기본값: 전경색 흰색, 배경색 검은색

### 2.7 커서

- 형태: 글자가 그려지는 **칸을 꽉 채우는 블록**으로 표시한다.
  - 채우는 색은 배경색과 전경색을 일정 비율로 섞은, 배경에 가까운 중간색이다.
  - 혼합 비율(배경:전경)의 기본값은 **6:4** 이며, 아래 API 로 바꿀 수 있다.

    ```cpp
    void set_cursor_blend(int bg_weight, int fg_weight);  // 배경:전경 가중치 (기본 6:4)
    ```

    - 가중치는 합으로 정규화하므로 `(6, 4)` 와 `(60, 40)` 은 같다. 합이 0 이면 무시한다.
  - 칸에 이미 글자가 있으면(방향키/Home 등으로 기존 입력 위에 커서가 온 경우) 블록 위에 전경색으로 글자를 다시 그려 가려지지 않게 한다. 한글 모드의 2칸 블록이 다음 칸 글자(또는 한글의 왼쪽 절반)까지 덮을 때도 그 글자들을 모두 다시 그려 가려지지 않게 한다.
  - **깜빡임**: 블록이 시스템 캐럿 깜빡임 속도(`GetCaretBlinkTime`)로 깜빡인다. 타이핑·커서 이동·출력 직후에는 즉시 보이게 한 박자 유지한다. 깜빡임 간격은 아래 API 로 바꿀 수 있다.

    ```cpp
    void set_cursor_blink(int interval_ms);  // 토글 간격(ms). 0 이면 깜빡임 없이 항상 표시
    ```

    - 기본값은 시스템 캐럿 깜빡임 속도를 따른다. (시스템에서 깜빡임을 꺼 두었으면 깜빡이지 않고 항상 표시한다.)
- 커서 위치는 자식이 VT 시퀀스(CUP 등)로 제어하며, 화면 내 어디로든 이동할 수 있다. 자식이 커서 표시/숨김(DECTCEM `?25`)을 보내면 따른다(숨김이면 블록을 그리지 않는다).
- **IME 조합 중 표시**(조합 커서 이원화·조합 글자 인라인 렌더)는 ConBox 가 직접 그리는 단계(이후 M5)에서 다룬다. 현재는 시스템 기본 인라인 표시에 맡긴다.
- **IME 입력 상태에 따른 커서 폭**: 현재 IME가 한글(자모) 입력 모드이면 커서를 한글 한 글자 폭(영문 2배)의 **2칸 블록**으로, 영문 모드이면 **1칸 블록**으로 표시한다.
  - 자음을 쳐서 조합이 시작되기 전이라도 한/영 상태만으로 폭이 결정된다.
  - 한/영 키로 모드를 바꾸면 즉시 반영된다 (별도 키 입력 불필요).

### 2.8 입력 동작

ConBox 는 입력 싱크가 걸린 **터미널(raw) 모드**로 동작한다(로컬 줄편집 없음). 키 입력을 로컬에서
편집하지 않고 VT 시퀀스/UTF-8 로 인코딩해 싱크(자식 stdin)로 보내며, 화면(에코·커서·줄편집)은
자식이 VT 출력으로 그린다. 키 → 자식 바이트 매핑은 **§3.2** 를 따른다.

- `Ctrl+Home`/`Ctrl+End`/`PageUp`/`PageDown` 은 스크롤백 보기를 위/아래로 이동하는 로컬 동작이다. 그 밖의 키는 모두 자식으로 보낸다.
- 한글 IME: 확정분만 UTF-8 로 자식에 보낸다. 조합 종료 트리거 키(방향키/Home/End/Delete/Enter/Tab/Esc 등) 직전에 조합을 강제 확정해 "제자리 완성 후 트리거" 순서를 보장한다(§3.1, 실타이핑 수동 검증 대상). 조합 중 인라인 표시는 여전히 시스템 기본에 맡긴다.
- (단독 텍스트 위젯의 로컬 줄편집 / `on_enter` 확정 콜백 / Shift+Enter / 한글 조합 삽입은 셀 그리드·터미널 모델로 흡수되며 제거되었다.)

### 2.9 클립보드 및 선택

- `Ctrl+V` / 우클릭 붙여넣기는 클립보드 텍스트를 자식 stdin 으로 보낸다(§3.2). `Ctrl+C` 는 자식에 인터럽트(`0x03`)를 보낸다.
- 마우스 드래그 선택·더블클릭 단어 선택·자동 복사는 셀 그리드 좌표 기준으로 다시 구현하는 단계(이후 M3a-2)에서 복구한다. 현재는 미구현이다.

### 2.10 스크롤

- 마우스 휠로 **스크롤백**(과거 출력)을 위로 볼 수 있다. 새 출력이 오면 맨 아래로 복귀한다.
- `print()` 로 새 출력이 오면 강제로 맨 아래로 스크롤한다.
- 세로 스크롤바는 스크롤백이 있을 때만(내용이 화면을 넘을 때만) 표시한다.

### 2.11 참조

- 명세에 없는 세부 동작은 **Windows Terminal (wt.exe)** 을 참조한다.

### 2.12 검증용 데모

- ConBox 소스코드의 이식성 확보를 위해 데모 기능은 ConBox 소스코드와 **분리**하여 구현한다.
- `Project/` 폴더의 MFC 대화상자 기반 프로젝트를 사용한다.
- ConBox는 메인윈도우(`CConBoxDlg`)의 차일드 윈도우로 생성한다.
  - 확인/취소 버튼은 두지 않고, ConBox가 클라이언트 영역을 사방 **20px 마진**으로 꽉 채운다.
  - 타이틀바에 최소화/최대화 버튼을 둔다(최소화로 작업표시줄 아이콘화). 창 크기 조절·최대화 시 ConBox도 함께 리사이즈된다.
  - 시작 시 현재 폰트 기준으로 ConBox 가 **96×32 칸(영문 기준)** 이 되도록 메인 창 크기를 잡고 화면 작업 영역 중앙에 띄운다. (그리드 픽셀은 `client_size_for_grid` 로 구하고, 데모 마진·세로 스크롤바 폭·비클라이언트 영역을 더해 창 크기를 정한다.)
- 데모 앱은 **System DPI Aware** 로 동작한다(비100% 배율에서 글자가 선명). DPI 인식은 프로세스 단위 속성이라 ConBox 컨트롤이 아니라 호스트(데모) 앱에서 설정한다. (방법은 개발노트 참고)

#### 초기 표시 내용

- 데모는 시작 시 ConBox의 개요와 특징을 소개하는 도표 화면을 표시한다.
- 도표의 구체적 내용(로고/문구/테두리 등)은 ConBox의 기능 요구가 아니라 데모의 표현물이므로, 단일 출처인 데모 코드(`Project/ConBoxDlg.cpp`)를 따른다.

## 3. ConExe

ConExe 는 ConPTY(의사 콘솔)로 터미널 기반 자식 프로그램을 실행하고, 그 콘솔 입출력을
생성 시 지정된 ConBox 인스턴스에 연결하는 헬퍼 클래스다. 자식이 진짜 콘솔로 인식하므로
라인 지향 도구부터 풀스크린 TUI(claude.exe, vim, htop 등)까지 한 경로로 동작한다.
상세 항목은 단계적으로 채우며, 단계별 로드맵·진행상황은 `Documents/개발계획서.md` 를 따른다.

### 3.1 입력기(IME) 처리 — 한글 조합 합격 기준 [필수]

ConExe 에 연결된 ConBox 가 raw 입력을 자식에 전달할 때, 한글 IME 조합과 키 입력 사이의
순서를 다음과 같이 보장한다. 이는 풀스크린 TUI 위에서 한글을 칠 때 흔히 깨지는 동작을
바로잡기 위한 것으로, **claude.exe 뿐 아니라 vim 등 다른 TUI 에서도 동일하게** 적용한다.

- **조합 종료 트리거의 순서 보장**: 한글 조합(미완성) 중에 조합을 끝내는 키
  (방향키 좌/우/위/아래, Home, End, Delete, Enter, Tab, Esc, 마우스 클릭 등)가 들어오면,
  ConBox 는 **먼저** 조합을 강제 확정(`ImmNotifyIME(NI_COMPOSITIONSTR, CPS_COMPLETE)`)하여
  완성된 글자의 UTF-8 바이트를 자식 stdin 으로 보내고, **그 다음에** 그 트리거 키의 VT
  시퀀스를 보낸다.
  - 합격 기준: 한글을 조합하다가 방향키를 누르면, 완성 글자가 **조합하던 자리에서 완성된 뒤**
    커서가 이동해야 한다. (옮겨진 칸에서 완성되면 불합격.) 즉 자식이 받는 바이트 순서는 항상
    `[완성 글자][트리거 키]` 여야 하며, `[트리거 키][완성 글자]` 가 되어선 안 된다.
- **조합 중 표시(로컬)**: 조합 중인(미완성) 글자는 ConBox 가 커서 칸에 로컬로 표시하고
  (조합 중 속 빈 테두리 커서 스타일 재활용), **확정된 분량만** 자식에 전송한다. 미완성
  조합 바이트를 자식에 흘려보내지 않는다.
- **검증**: 한글 키보드가 필요하므로 직접 타이핑으로 확인한다(개발노트의 수동 검증 항목).
  claude.exe·vim 양쪽에서 "조합 중 방향키/Home/End/클릭 → 제자리 완성 후 이동" 을 확인한다.

  (이 기준은 raw IME 처리를 다루는 단계(개발계획서 M5)에서 구현·검증하며, 그 전 단계의 입력은
  완성형 글자만 우선 전달하는 축소 형태일 수 있다. 최종 합격 기준은 위와 같다.)

### 3.2 공개 API 와 입출력 연결 (M1·M2·M3)

```cpp
void attach(CConBox* box);                       // 출력 + 입력 + 리사이즈 연결(양방향)
bool start(const char* cmdline, int cols, int rows);  // 자식을 ConPTY 로 실행 (cmdline 은 UTF-8)
void write(const char* data, int len);           // 자식 stdin 으로 바이트열 전송
void resize(int cols, int rows);                 // 자식 콘솔 크기 변경 (ResizePseudoConsole)
void stop();                                     // 자식/PTY/파이프/타이머 정리 (여러 번 호출 안전)
bool is_running() const;                          // 자식이 실행 중이고 출력 경로가 살아 있는지
void set_exit_callback(void (*cb)(void* user), void* user);  // 자식 자연 종료 시 1회 호출
```

- `attach(box)` 는 **출력**(자식 콘솔 출력 → ConBox), **입력**(ConBox 키 입력 → 자식 stdin),
  **리사이즈**(ConBox 그리드 크기 변경 → 자식 콘솔)까지 한 번에 연결한다. 내부적으로 ConBox 의
  입력 싱크·리사이즈 싱크(아래)에 자신을 등록한다.
- `start` 는 입력/출력 파이프 쌍과 의사 콘솔(ConPTY)을 만들고 자식을 실행한다. 출력은 내부
  타이머가 주기적으로 폴링해 ConBox `print()` 로 전달하며, ConBox 가 VT 시퀀스를 해석한다(M3).
- `resize` 는 `ResizePseudoConsole` 로 의사 콘솔 크기를 바꿔 자식이 화면을 다시 그리게 한다.
  ConBox 의 리사이즈 싱크에 연결되어, 창 크기 변경으로 그리드가 바뀌면 자동으로 호출된다.
- `cols/rows` 는 의사 콘솔 크기이며 보통 ConBox 그리드(`grid_cols()`/`grid_rows()`)에 맞춘다.
- `set_exit_callback` 은 자식이 **자연 종료**(예: 셸 `exit`)했을 때 한 번 호출되는 콜백을 등록한다.
  콜백 시점에는 이미 정리(stop)가 끝나 `is_running()` 이 false 다(콜백 안에서 새 `start()` 로 재시작
  안전). 명시적 `stop()` 으로 끝낸 경우엔 호출되지 않는다. ConPTY 에서는 자식이 끝나도 conhost 가
  출력 파이프를 쥐고 있어 파이프 EOF 로는 종료를 못 보므로, **자식 프로세스 핸들을 폴링**해 감지한다.

#### ConBox 입력 싱크(터미널/raw 모드)

```cpp
void set_input_sink(void (*sink)(const char* bytes, int len, void* user), void* user);
```

- 싱크가 설정되면 ConBox 는 **터미널(raw) 모드**로 동작한다. 키 입력을 로컬에서 편집하지 않고
  (로컬 에코 끔), 키/문자를 VT 시퀀스와 UTF-8 바이트로 인코딩해 싱크로 흘려보낸다. 화면(에코·
  커서·줄편집)은 자식이 VT 출력으로 그린다. `nullptr` 을 주면 읽기 전용 뷰어 모드로 돌아간다.
- ConExe 는 `attach()` 에서 이 API 로 자신을 등록한다(호스트가 직접 부를 필요 없음).

#### ConBox 리사이즈 싱크

```cpp
void set_resize_sink(void (*sink)(int cols, int rows, void* user), void* user);
```

- 창 크기 변경 등으로 ConBox 화면 그리드(cols/rows)가 바뀌면 새 크기를 이 콜백으로 통지한다.
  ConExe 는 `attach()` 에서 등록해 `resize()`(→`ResizePseudoConsole`)로 연결한다.

#### 키 입력 → 자식 전송 매핑 (터미널 모드)

| 입력 | 자식으로 보내는 바이트 |
|------|------------------------|
| 인쇄 가능 문자 / 한글(완성·IME 확정분) | UTF-8 |
| Enter | `\r` (0x0D) |
| Backspace | DEL (0x7F) |
| Tab / Esc / Ctrl+영문 | 해당 제어코드(0x09 / 0x1B / 0x01~0x1A) |
| 방향키 ←↑→↓ | 일반: `ESC[D`/`A`/`C`/`B`, 앱 커서 키(DECCKM `?1h`): `ESC O D`/`A`/`C`/`B` |
| 수정자 + 방향키/Home/End | `ESC[1;{mod}{문자}` (mod = 1 + Shift·1 + Ctrl·4 의 +1, 예 Shift=2·Ctrl=5) |
| Home / End / Delete / Insert | `ESC[H` / `ESC[F` / `ESC[3~` / `ESC[2~` |
| PageUp / PageDown | `ESC[5~` / `ESC[6~` |
| Shift+Tab | `ESC[Z` (backtab) |
| F1~F4 / F5~F12 | `ESC O P`~`S` / `ESC[15~`·`17~`…`24~` |
| Ctrl+C | 인터럽트 `0x03` (선택 텍스트 복사는 M3a-2 에서 복구) |
| Ctrl+V / 붙여넣기 | 클립보드 텍스트를 자식 stdin 으로 전송. bracketed paste(`?2004h`)면 `ESC[200~`…`ESC[201~` 로 감싼다 |

- 자식이 보내는 모드 시퀀스에 따라 입력 인코딩이 바뀐다: 앱 커서 키(DECCKM `?1`)는 방향키 접두를
  `ESC[` → `ESC O` 로, bracketed paste(`?2004`)는 붙여넣기를 마커로 감싼다.
- 한글 IME: 터미널 모드에서는 **확정분만** UTF-8 로 자식에 보낸다. 조합 종료 트리거 키 직전에
  강제 확정해 순서를 보장한다(§3.1, 실타이핑 수동 검증 대상).

#### 현재 지원 범위 / 한계

- 자식 콘솔 출력의 VT 시퀀스를 ConBox 가 셀 그리드 위에서 해석한다: 커서 이동·ED/EL·SGR
  16/256/트루컬러·자동 줄바꿈·DECTCEM `?25`(M3) + **대체 화면·스크롤 영역·줄/문자 삽입삭제·
  DSR/DA 응답**(M4·M5). 창 리사이즈는 `ResizePseudoConsole` 로 동기화된다. 풀스크린 TUI(vim·
  claude.exe 등)가 ConBox 안에서 구동된다.
- 남은 항목: 한글 IME 조합 순서(§3.1)는 코드 구현 완료·실타이핑 수동 검증 대상(M5), 마우스
  선택/복사는 M3a-2 미구현. 진행 단계는 `개발계획서.md`.
