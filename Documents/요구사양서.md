# 소프트웨어 요구사항 명세서 (SRS)

※ 본 문서에서 표현되는 상대 경로는 모두 프로젝트 루트를 기준으로 한다.

---

## 1. 프로젝트 개요

- 타 프로젝트에 쉽게 이식 가능한 커스텀 윈도우 컨트롤 **ConBox**를 제작한다.
- ConBox는 wt.exe처럼 텍스트 입력과 출력을 모두 처리하는 터미널 컨트롤이다.
- MFC 기반 CWnd 파생 클래스로 구현한다.
- 모든 문자열은 **UTF-8 기반**이며, 문자열 입력 API는 `const char*`를 사용한다.
  - (C++ 문자열 리터럴을 그대로 넘길 수 있도록 `char*`가 아닌 `const char*`로 한다.)

---

## 2. 창 생성 API

```cpp
void open(CWnd* parent, int x0, int y0, int x1, int y1);
```
- 리소스 에디터 없이 좌표 지정만으로 창을 생성한다.

---

## 3. 폰트 설정 API

```cpp
void set_efont(const char* 폰트이름, float 크기, const char* 추가설정);  // 영문 폰트
void set_kfont(const char* 폰트이름, float 크기, const char* 추가설정);  // 한글 폰트
```

- 크기 단위: **포인트(pt)**. 소수점 크기(예: 10.5, 12.1)도 지정할 수 있다.
  - 다만 폰트는 최종적으로 정수 픽셀 높이로 래스터화되므로, 소수 크기는 어느 정수 픽셀에 떨어질지를 더 세밀하게 고르는 의미이며 같은 DPI에서 일부 값은 같은 픽셀이 될 수 있다.
- 기본값
  - 영문: Consolas, 13pt, Bold
  - 한글: 맑은 고딕, 13pt, Bold

### 추가설정 문자열 형식

LOGFONT 속성을 간결한 문자열로 지정한다. 숫자가 알파벳 앞에 오면 해당 속성의 정도(값)를 의미한다.

| 기호 | 속성 | 값 필요 여부 | 예시 |
|------|------|-------------|------|
| `B`  | Bold | 아니오 | `"B"` |
| `I`  | Italic | 아니오 | `"I"` |
| `U`  | Underline | 아니오 | `"U"` |
| `S`  | Strikeout | 아니오 | `"S"` |
| `W`  | Weight | 예 (앞에 숫자) | `"700W"` |

- 조합 예시: `"BI"` (Bold+Italic), `"400W"` (Weight=400), `"BU"` (Bold+Underline)

### 줄 높이를 영문 기준으로 맞추는 옵션

```cpp
void set_kfont_match_efont(bool on);   // 기본값: 켜짐(true)
```

- 켜지면(기본) 한글을 영문과 어울리게 자동 보정한다.
  - `set_kfont` 의 **크기(pt)는 무시**하고, 한글을 영문과 **같은 픽셀 높이**로 맞춘다. (단순히 같은 pt 로는 한글이 더 커 보이므로 실제 글자 높이가 같아지도록 보정한다.)
  - 줄 높이를 영문 기준으로 잡아 박스 그리기 문자(┌─┼ 등) 표가 세로로 촘촘히 이어진다.
  - 한글 자간(2칸 폭 안의 빈 공간)이 넓어 보이는 문제를 줄이기 위해, 한글은 가로로 찌그러뜨리지 않고(자연 폭 유지) 영문 칸 폭을 그 한글 폭에 맞춰 약간 좁힌다(영문 장평). 그 결과 한글이 두 칸을 거의 채워 자간이 좁아진다.
- 끄면 한글은 `set_kfont` 로 지정한 크기 그대로이고, 영문도 원본 폭이며, 줄 높이는 두 폰트 중 큰 쪽이 된다.

## 4. 문자 렌더링

- 가변폭 렌더링 금지 — 모든 문자는 **고정 그리드** 위치에 그린다.
- 한글은 영문의 **2배 폭**으로 렌더링한다.
- 폰트 또는 창 크기 변경 시 가로/세로 표시 가능 문자 수를 재계산한다.
- 가로폭 초과 시 **자동 줄바꿈** 처리한다.
- 세로 줄 수가 창 높이 초과 시 **세로 스크롤바** 표시, 미만이면 숨긴다.

### 안쪽 여백 (set_margin)

```cpp
void set_margin(int top, int left = -1, int bottom = -1, int right = -1);
```

- 클라이언트 영역 안쪽 여백(픽셀)을 둔다. 여백 안쪽에만 글자가 그려진다.
- 기본값은 사방 **10px**.
- CSS 단축 스타일로 생략한 변은 다른 변을 따른다(순서대로 평가).
  - `left` 가 음수면 `top` 을 따른다.
  - `bottom` 이 음수면 `top` 을 따른다.
  - `right` 가 음수면 (위에서 결정된) `left` 를 따른다.
  - 예: `set_margin(5)` → 사방 5, `set_margin(5, 8)` → 상하 5 / 좌우 8.
- `open` 이후 호출하면 창 크기 변경과 동일하게 가로/세로 표시 가능 문자 수를 재계산한다.

## 5. 내부 버퍼

- 논리 줄 단위로 관리한다 (Enter 기준, Shift+Enter는 같은 논리 줄).
- 문자 버퍼와 **문자 단위 색상 버퍼**를 함께 관리한다.
- 화면에 표시되는 시작 줄과 끝 줄 인덱스를 보유한다.

---

## 6. 출력 API

```cpp
void print(const char* text);
```

- 현재 커서 위치에서 이어서 출력하며, 항상 버퍼 끝에 추가된다.
- 특수문자 처리:
  - `\n` : 다음 줄 맨 처음으로 이동
  - `\r` : 현재 줄 맨 처음으로 이동 (덮어쓰기)
  - `\t` : 탭. 다음 4의 배수 칸(탭 스톱)까지 공백으로 이동한다. (예: 현재 2칸이면 2칸, 4칸이면 4칸 이동)
- 새 내용 추가 시 스크롤이 **강제로 맨 아래로** 이동한다.

---

## 7. 색상 API

```cpp
void set_color(COLORREF fg);         // 전경색(글자색) 설정
void set_bg_color(COLORREF bg);      // 배경색 설정
```

- 설정 이후 호출되는 `print()`에 적용된다.
- 기본값: 전경색 흰색, 배경색 검은색

## 8. 커서

- 형태: 글자가 그려지는 **칸을 꽉 채우는 블록**으로 표시한다.
  - 채우는 색은 배경색과 전경색을 일정 비율로 섞은, 배경에 가까운 중간색이다.
  - 혼합 비율(배경:전경)의 기본값은 **6:4** 이며, 아래 API 로 바꿀 수 있다.

    ```cpp
    void set_cursor_blend(int bg_weight, int fg_weight);  // 배경:전경 가중치 (기본 6:4)
    ```

    - 가중치는 합으로 정규화하므로 `(6, 4)` 와 `(60, 40)` 은 같다. 합이 0 이면 무시한다.
  - 칸에 이미 글자가 있으면(방향키/Home 등으로 기존 입력 위에 커서가 온 경우) 블록 위에 전경색으로 글자를 다시 그려 가려지지 않게 한다. 한글 모드의 2칸 블록이 다음 칸 글자(또는 한글의 왼쪽 절반)까지 덮을 때도 그 글자들을 모두 다시 그려 가려지지 않게 한다.
  - **깜빡임 없음**: 정적 블록으로 표시한다. (깜빡임 타이머를 두지 않아 성능에 유리하다.)
- **IME 조합 중 커서 모양(이원화)**: IME가 한글 모드이고 조합(미완성) 중일 때는 블록을 채우지 않고, 조합 중인 글자 영역을 **두께 2px의 속이 빈 사각형**(커서색 테두리)으로 감싼다. 속은 배경색이 보이고 조합 글자는 본문과 같은 색으로 표시된다. 조합 중이 아니거나 영문 상태이면 위의 꽉 찬 블록 커서를 쓴다.
- 커서는 **입력 영역 내에서만** 이동한다 (이미 출력된 영역으로 이동 불가)
- **IME 입력 상태에 따른 커서 폭**: 현재 IME가 한글(자모) 입력 모드이면 커서를 한글 한 글자 폭(영문 2배)의 **2칸 블록**으로, 영문 모드이면 **1칸 블록**으로 표시한다.
  - 자음을 쳐서 조합이 시작되기 전이라도 한/영 상태만으로 폭이 결정된다.
  - 한/영 키로 모드를 바꾸면 즉시 반영된다 (별도 키 입력 불필요).

## 9. 입력 동작

### 편집 범위
- 이미 출력된 내용은 **편집 불가**
- 현재 입력 줄만 수정 가능

### 키 동작

| 키 | 동작 |
|----|------|
| `Enter` | 입력 확정, 버퍼에 추가, 다음 줄 이동, 콜백 발생 |
| `Shift+Enter` | 화면상 줄바꿈만 (미확정, 버퍼 미추가) |
| `Backspace` / `Delete` | 입력 영역 내에서만 동작 |
| `방향키 좌/우` | 입력 영역 내에서만 커서 이동 |
| `방향키 위/아래` | 여러 화면 줄에 걸친 입력에서 위/아래 화면 줄로 이동. 대상 줄 끝보다 가로 위치가 뒤면 그 줄 끝으로 보정한다. 가로 위치는 끈적(sticky)하게 처음 세로 이동 시점의 칸을 유지하며, 좌/우 이동·타이핑 시 초기화된다. 현재 입력 줄(편집 영역) 안에서만 이동(확정된 윗줄로는 못 감). |
| `Tab` | 다음 4의 배수 칸(탭 스톱)까지 이동 |
| `Home` / `End` | 현재 줄 시작 / 끝 |
| `Ctrl+Home` / `Ctrl+End` | 버퍼 전체 시작 / 끝으로 스크롤 |

### 한글 입력
- Windows IME 표준 방식으로 자연스럽게 조립 과정을 표시한다.
- 줄 중간에 커서를 두고 조합을 시작하면, 조합 글자가 **삽입(insert)** 되어 뒤 글자들이 즉시 오른쪽으로 밀린다(덮어쓰지 않는다). 조합 취소 시 원래대로 복구된다.

### Enter 콜백

```cpp
void (*on_enter)(const char* input);
```

- Enter 확정 시 한 번 호출된다.
- `input`에는 Shift+Enter로 입력된 줄을 포함한 전체 텍스트가 `\n` 구분자와 함께 전달된다.

## 10. 클립보드 및 선택

- 마우스 드래그로 텍스트 선택 (출력 영역 포함)
- **더블클릭**: 그 위치의 단어를 선택해 자동으로 클립보드에 복사한다. 단어 경계는 공백(스페이스/탭/줄바꿈)으로 끊되, 추가로 **영문 / 한글 / 특수문자**가 섞이지 않도록 같은 부류가 연속된 구간만 한 단어로 본다. (숫자는 영문 부류로 묶고, 연속 특수문자는 한 단어로 묶는다. 예: `abc한글!!!` → `abc` / `한글` / `!!!`)
- 선택된 텍스트는 자동으로 클립보드에 복사
- 우클릭: 커서 위치에 붙여넣기
- `Ctrl+C`: 복사, `Ctrl+V`: 붙여넣기

## 13. 검증용 데모

- ConBox 소스코드의 이식성 확보를 위해 데모 기능은 ConBox 소스코드와 **분리**하여 구현한다.
- `Project/` 폴더의 MFC 대화상자 기반 프로젝트를 사용한다.
- ConBox는 메인윈도우(`CConBoxDlg`)의 차일드 윈도우로 생성한다.
  - 확인/취소 버튼은 두지 않고, ConBox가 클라이언트 영역을 사방 **20px 마진**으로 꽉 채운다.
  - 타이틀바에 최소화/최대화 버튼을 둔다(최소화로 작업표시줄 아이콘화). 창 크기 조절·최대화 시 ConBox도 함께 리사이즈된다.
- 데모 앱은 **System DPI Aware** 로 동작한다(비100% 배율에서 글자가 선명). DPI 인식은 프로세스 단위 속성이라 ConBox 컨트롤이 아니라 호스트(데모) 앱에서 설정한다. (방법은 개발노트 참고)
