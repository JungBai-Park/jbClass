# 설계명세서

코드를 처음 보는 사람(또는 새 세션)이 resume 없이도 구조를 빠르게 파악하고 손댈 수 있도록 정리한다.
세부 구현은 `Source/` 의 코드가 정본이며, 이 문서는 큰 그림과 확장 지점을 설명한다.

## 1. ConBox

### 1.1 구성

- `Source/ConBox.h` / `ConBox.cpp` — 컨트롤 본체 `CConBox` (CWnd 파생). **이식 단위는 이 두 파일뿐**이다. 파일 내부 전용 보조 함수(`IsWideChar`, 폰트 opts 파서 `ParseFontOpts`, 단어 분류 `CharClass`, 박스/블록 직접 렌더 `DrawBlockElement`/`DrawBoxLine`)는 `ConBox.cpp` 상단의 `static` 함수로 둔다.
- `Project/` — 검증용 MFC 대화상자 데모. `CConBoxDlg` 가 `CConBox` 를 자식으로 띄운다. 컨트롤과 분리되어 있다.

자세한 빌드/관례는 `개발노트.md` 참고.

### 1.2 데이터 모델 (셀 그리드)

```
screen : vector<Row>                // 현재 화면. 항상 rows 개 줄, 각 줄 cols 칸 셀(고정 폭)
  Row = vector<CharInfo>
    CharInfo { wchar_t ch; COLORREF fg, bg; bool wide; }
      - wide=true : 2배 폭 글자(한글/CJK)의 lead 칸. 바로 다음 칸은 trail(ch=0, 렌더 생략)
      - ch==L' ' : 빈 칸
scrollback : vector<Row>            // 화면 위로 밀려난 줄들(상한 MAX_SCROLLBACK 넘으면 앞부터 버림)
main_saved : vector<Row>            // 대체 화면(alt screen) 진입 시 백업해 둔 주 화면(복귀 시 swap)
```

상태 변수:
- `cur_row` / `cur_col` — 커서. **화면 내 0-based 칸 좌표**(행 0..rows-1, 칸 0..cols). VT 시퀀스가 직접 옮긴다.
- `view_top` — 보기 맨 위. (scrollback + screen) 통합 인덱스. 평상시 맨 아래(`scrollback.size()`).
- `cursor_visible` — 자식의 DECTCEM(`?25`) 커서 표시 여부(기본 true). `saved_row` / `saved_col` — DECSC/DECRC 저장 커서.
- `scroll_top` / `scroll_bot` — 스크롤 영역(DECSTBM). LF/RI/IL/DL/SU/SD 가 존중하는 줄 범위. 기본 화면 전체 `[0, rows-1]`. `reset_screen` 이 rows 변동 시 범위를 clamp(깨지면 전체로 복원).
- `alt_active` — 대체 화면 활성 여부. 활성이면 `main_saved` 에 주 화면이 백업돼 있고 스크롤백은 동결(보이지 않음, `update_scrollbar`/휠/스크롤바가 무시). `saved_main_row` / `saved_main_col` — 진입 시 저장한 주 화면 커서.
- `cols` / `rows` — 화면 가로 칸 / 세로 줄 수.
- `cell_w` / `cell_h` / `cell_base` — 고정 그리드 한 칸 크기와 베이스라인(픽셀).
- `kfont_match_efont`, `kfill_ratio` / `emin_ratio` — 한글 높이 맞춤·장평 비율. `calc_cell_size` 가 사용(아래 폰트 항 참고).
- `cursor_on` / `cursor_blink_ms` — 커서 깜빡임 표시 상태와 토글 간격(ms). 0 이면 깜빡이지 않고 항상 표시. 기본 간격은 `GetCaretBlinkTime`.
- `efont_lf` / `kfont_lf` — 사용자가 지정한 폰트 원본 LOGFONT. 장평/높이 보정은 폰트 객체에만 적용하고 원본은 보존한다.
- `margin_top` / `margin_bottom` / `margin_left` / `margin_right` — 클라이언트 안쪽 여백(픽셀, 기본 10). `update_metrics` 가 여백을 뺀 영역으로 cols/rows 를 계산하고, 모든 픽셀 변환이 좌/상 여백을 더한다.
- `glyph_level` — 박스/블록 직접 렌더 수준(0 끔 / 1 블록 요소 / 2 +박스선, 기본 1). `OnPaint` 가 참조.
- VT 파서 상태: `vt_state`(GROUND/ESC/CSI/OSC), `vt_params` / `vt_nparam` / `vt_priv`(CSI `?` DEC 프라이빗 마커) / `vt_gtlt`(CSI `<`/`=`/`>` 접두 마커 — 2차 DA·kitty keyboard·XTMODKEYS 등, `dispatch_csi` 가 통째 무시해 최종 바이트를 표준 명령으로 오인하지 않게 함). `print` 호출(청크) 사이에 유지된다.
- SGR 현재 속성: `cur_fg` / `cur_bg`(색), `cur_bold`(밝은색 매핑), `cur_reverse`(전경/배경 스왑). `put_char` 가 셀에 반영.
- 입력 모드(자식이 DEC 프라이빗 모드로 켬): `app_cursor_keys`(DECCKM `?1`, 방향키를 `ESC O x` 로), `bracketed_paste`(`?2004`, 붙여넣기를 `ESC[200~`…`ESC[201~` 로 감쌈). `terminal_keydown`/`paste_clipboard` 가 참조.
- `back_dc` / `back_bmp` / `back_old_bmp` / `back_w` / `back_h` — 더블 버퍼링 캐시(`ensure_back_buffer`).

#### 좌표계 (칸 = 시각 칸)
- **셀 그리드 좌표** `(cur_row, cur_col)` — 화면 내 칸. 2배 폭 글자는 lead+trail 두 칸을 차지하므로 칸 = 시각 칸이다.
- **통합 인덱스** `idx` — scrollback 다음에 screen 이 이어지는 전체 줄 인덱스. `line_at(idx)` 가 해당 줄을 돌려준다. 총 줄 수 = `scrollback.size() + rows`.
- **화면/픽셀 좌표** — 보기 맨 위 `view_top` 기준 줄 `row` 와 픽셀(좌/상 여백 포함).

### 1.3 주요 흐름

#### 출력 `print(const char* utf8)` → VT 파서
UTF-8 → UTF-16 변환 후 각 글자를 **`vt_feed(wc)` 상태기계**에 넣는다(파서 상태는 멤버라 ConExe 청크 호출 사이에 유지). 끝에 맨 아래로 스크롤·스크롤바·커서 bump·다시 그림.
- `vt_feed` 상태: **GROUND**(글자→`put_char`, C0 제어 CR/LF/BS/TAB(8의 배수)/BEL, ESC→ESC) / **ESC**(`[`→CSI, `]`→OSC, `7`/`8`=DECSC/DECRC, `M`=RI — 영역 상단이면 `scroll_down_region`) / **CSI**(파라미터 누적, `?`=프라이빗, `<`/`=`/`>`=접두 마커 `vt_gtlt`, 최종 바이트→`dispatch_csi`) / **OSC**(BEL 또는 ST 까지 버림 — 창 제목 등).
- `dispatch_csi(fin)`: 맨 앞에서 `vt_gtlt` 면(프라이빗 `<>=` 시퀀스) 통째 무시. 그 외 커서 이동(CUU/CUD/CUF/CUB·CUP/HVP·CHA·VPA), 지우기(ED `J` / EL `K` — `erase_cells`)·문자 지우기(ECH `X`), SGR(`m`), 모드 set/reset(`h`/`l`: DECTCEM `?25` / 대체화면 `?1049`·`?1047`·`?47` / `?1048` 커서 / DECCKM `?1` / bracketed `?2004`), 커서 저장·복원(`s`/`u`), **풀스크린**(스크롤 영역 DECSTBM `r`, 줄 삽입삭제 IL `L`/DL `M`, 문자 삽입삭제 ICH `@`/DCH `P`, 영역 스크롤 SU `S`/SD `T`), **질의 응답**(DSR `n` → CPR/상태를 입력 싱크로, DA `c` → VT102).
- SGR: `0` 리셋 / `1`·`22` bold / `7`·`27` reverse / `30-37`·`90-97` fg / `40-47`·`100-107` bg / `39`·`49` 기본색 / `38`·`48` 의 `5;n`(256색)·`2;r;g;b`(트루컬러). 256색→COLORREF 는 static `Xterm256ToRgb`(16 기본 + 6³ 큐브 + 24 그레이).
- `put_char(wc)`: 칸 폭 초과면 autowrap(`cur_col=0` + `line_feed`), 셀에 현재 색(reverse 면 스왑) 기록, 2배 폭은 다음 칸을 trail(ch=0)로. `line_feed`: 커서가 영역 하단(`scroll_bot`)이면 `scroll_up_region(1)`, 아니면 한 줄 내림.
- **스크롤/삽입삭제 일꾼**: `scroll_lines_up/down(top,bot,n)` 가 줄 범위 안 순수 회전(scrollback 미관여). `scroll_up_region` 은 주 화면 + 영역 상단 0 일 때만 밀려난 줄을 scrollback 으로 보존(아니면 버림), `scroll_down_region` 은 RI/SD. IL/DL 은 `scroll_lines_down/up(cur_row,scroll_bot,n)`, ICH/DCH 는 줄 안에서 `insert`/`erase` 후 `resize(cols)`. 대체 화면은 `enter/leave_alt_screen`(주 화면 `main_saved` 와 swap, 빈 화면/커서/영역 리셋, 복귀 시 `reset_screen`).

#### 입력 (터미널 모드 전용)
- `OnChar` — 입력 싱크가 있으면 문자/제어문자를 바이트로 인코딩해 싱크로(`send_input_bytes` / `send_input_wide`). 로컬 편집/에코는 하지 않는다.
- `OnKeyDown` — 입력 싱크가 있으면 `terminal_keydown` 이 방향키/편집키를 xterm VT 로 전송(인쇄 문자/Enter/Tab 등은 WM_CHAR 경로에 맡김). 방향키/Home/End 는 수정자(`ESC[1;{mod}x`)·앱 커서 키(`app_cursor_keys` → `ESC O x`)를 반영하고, Shift+Tab(`ESC[Z`)·F1~F12 도 인코딩한다. 붙여넣기는 `paste_clipboard`(bracketed paste 면 감쌈). 조합 종료 트리거 키(방향키/Home/End/Delete/Enter/Tab/Esc 등) 직전에 `finalize_composition` 으로 IME 조합을 강제 확정한다(§3.1). 싱크가 없으면(읽기 전용 뷰어, 대체 화면 제외) `Ctrl+Home/End`·`PageUp/Down` 스크롤만.
- `OnGetDlgCode` 이 DLGC_WANTALLKEYS 를 반환해 부모 대화상자가 키를 가로채지 않게 한다.

#### 리사이즈 동기화
`update_metrics` 가 cols/rows 를 다시 계산하고 `reset_screen` 으로 화면 그리드를 새 크기에 맞춘다(행 수/줄 폭 조정, 커서 clamp). cols/rows 가 **실제로 바뀌면** 리사이즈 싱크(`resize_sink`)로 새 크기를 통지한다 → ConExe `resize`(`ResizePseudoConsole`). `set_resize_sink` 로 등록한다(ConExe 가 `attach` 에서).

#### 폰트/셀 크기 `calc_cell_size()`
폰트 메트릭으로 고정 그리드 한 칸 크기 `cell_w`(가로) / `cell_h`(세로) / `cell_base`(베이스라인)를 정한다. DPI 는 `GetDeviceCaps(LOGPIXELSY)` 로 얻어 포인트→픽셀 환산에 쓴다(호스트가 DPI Aware 면 실제 DPI).

**공통 전처리**: 매번 영문을 원본(`efont_lf`)으로 되돌려 만든 뒤 자연 폭부터 측정한다(이전 호출의 장평 잔재 제거).
- `natw` = 영문 `"W"` 폭(`GetTextExtentPoint32`), `eh` = `tmHeight + tmExternalLeading`(영문 줄 높이), `ea` = `tmAscent`.

**`kfont_match_efont` ON (기본, set_kfont 크기 ≤ 0)** — 한글을 영문 높이에 맞추고 칸 폭을 한글 폭에서 역산한다.
1. **한글 높이 맞춤**: 한글을 영문 `lfHeight`(=em) 로 만들어 실제 높이 `kh0` 를 잰다. `kh0 != eh` 면 `newh = lfHeight · eh / kh0` 로 `lfHeight` 를 비례 보정해 다시 만든다(같은 em 만으로는 한글이 더 커 보이므로 실제 글자 높이를 맞춘다). 폭은 자연 폭(`lfWidth=0`) 유지 — 가로로 찌그러뜨리면 획이 깨진다.
2. **한글 폭 측정**: 높이 맞춘 한글 `"가"` 의 자연 폭 `kwid` 를 잰다.
3. **한 칸 폭 `cw` 결정** (장평의 핵심):
   - 1순위 후보: `cw = round(kwid / (2 · kfill_ratio))` — 한글 한 글자가 두 칸을 `kfill_ratio`(기본 0.92)만큼 채우도록 칸 폭을 한글 폭에서 역산.
   - 상·하한으로 자르기: `emin = round(natw · emin_ratio)`(기본 0.7). `cw` 를 `[emin, natw]` 로 clamp(최소 1). 즉 영문을 자연 폭보다 **넓히지 않고**(상한 natw), 가독성 하한(emin) 밑으로 **좁히지도 않는다**. 보통은 후보값이 범위 안이라 하한·상한이 발동하지 않는다.
   - 기본 폰트·DPI 96 실측 예: `natw=9`, `kwid=14` → 후보 `round(14/1.84)=8` → `[6, 9]` 범위 안 → `cw=8`.
4. **영문 장평**: 영문을 `lfWidth = cw` 로 다시 만들어 좁히고, 그 영문 `"W"` 실측 폭이 `cell_w` 가 된다.
5. `cell_h = max(eh, kh)`, `cell_base = max(ea, ka)`(거의 영문과 같다). → 한글은 왜곡 없이 자간만 좁아지고 줄 높이는 영문 기준으로 촘촘해진다.

**`kfont_match_efont` OFF (set_kfont 크기 > 0)**: 영문 원본 폭(`cell_w = natw`), 한글도 `kfont_lf` 원본 크기, `cell_h / cell_base = max(영문, 한글)`.

- 주의: GDI 객체를 DC 에 선택한 채로 `DeleteObject` 하면 안 되므로, 폰트 재생성 전 선택을 해제한다.
- 정수 변환 함정(검증 재현 시): 포인트→픽셀 `-(LONG)(pt·dpi/72 + 0.5)` 와 `cw=(int)(…+0.5)` 는 round-half-up, 높이 비례보정 `(LONG)(…)` 은 0 방향 절사. (개발노트 "폰트 메트릭 검증" 참고)

#### 그리기 `OnPaint()`
**더블 버퍼링**: 메모리 DC(`CreateCompatibleDC` + 호환 비트맵)에 한 프레임을 모두 그린 뒤 마지막에 `BitBlt` 로 화면에 복사한다(빠른 입력 시 깜빡임 제거). 이식성을 위해 일반 GDI 사용. 모든 픽셀 좌표는 좌/상 여백(`margin_left/top`)을 더한다.
순서: 배경 채움 → `view_top` 부터 rows 줄을 `line_at(idx)` 로 순회하며 칸 단위로 그림(칸 인덱스 = 가로 칸 좌표, 2배 폭 셀은 두 칸 그리고 다음 칸 건너뜀, `ch==0`(trail)·`L' '`(빈칸)은 글자 생략, 2배 폭은 kfont) → 커서. 베이스라인(TA_BASELINE) 정렬로 한·영 글자 바닥을 맞춘다. 블록/박스 문자는 `glyph_level` 에 따라 `DrawBlockElement`/`DrawBoxLine` 로 칸에 직접 칠한다.
커서는 칸을 꽉 채우는 **블록**으로, `cursor_on && cursor_visible` 일 때만 `get_cursor_rect()` 사각형을 `blend_cursor_color()`(배경:전경 = `cursor_bg_weight`:`cursor_fg_weight`, 기본 6:4, `set_cursor_blend` 로 변경)으로 채운다. 폭은 `is_hangul_mode()` 면 2칸·영문이면 1칸. 커서 블록이 덮는 글자는 블록 사각형으로 클리핑해 전경색으로 다시 그려 가려지지 않게 한다.
**깜빡임**: `WM_TIMER`(`CURSOR_TIMER`)의 `OnTimer` 가 `cursor_on` 을 토글하고 커서 칸만 `InvalidateRect`. 간격은 `cursor_blink_ms`(0 이면 항상 표시). 출력/입력 핸들러는 `bump_cursor()` 로 입력 직후 한 박자 보이게 한다. `cursor_visible` 이 false(자식이 DECTCEM `?25l` 로 커서를 숨김)면 블록을 그리지 않는다.

#### IME (한글)
터미널 모드 전용이라 조합 글자는 ConBox 가 직접 그리지 않고 시스템 기본 인라인 표시에 맡긴다(직접 조합 렌더는 향후). `OnImeStart` 가 조합/후보 창을 커서 위치로 옮기고, `OnImeComp` 는 확정분(GCS_RESULTSTR)만 `send_input_wide` 로 자식에 보낸다(미완성 조합은 흘려보내지 않음, `n==0` 빈 RESULTSTR 은 걸러 이중 전송 방지, `return 0` 으로 WM_CHAR 중복 차단). **조합 순서 보장(§3.1)**: `finalize_composition` 이 조합 중(`GCS_COMPSTR` 길이>0)이면 `ImmNotifyIME(NI_COMPOSITIONSTR, CPS_COMPLETE)` 로 강제 확정한다. CPS_COMPLETE 는 동기적으로 `WM_IME_COMPOSITION(GCS_RESULTSTR)` 을 디스패치하므로, 트리거 키 전송 전에 완성분이 먼저 나가 `[완성][트리거]` 순서가 보장된다(IME 가 스스로 먼저 확정하는 경우엔 no-op 안전망). 한/영 상태는 `is_hangul_mode()`(`ImmGetConversionStatus` 의 `IME_CMODE_NATIVE`)로 조회해 커서 블록 폭(한글 2칸/영문 1칸)을 정하고, 한/영 토글 시 오는 `OnImeNotify`(IMN_SETCONVERSIONMODE)가 커서 영역을 무효화해 즉시 반영한다.

#### 마우스/클립보드
마우스 드래그 선택·더블클릭 단어 선택·자동 복사는 셀 그리드 좌표로 재작성하는 단계(M3a-2)에서 복구하며 현재 미구현이다(관련 핸들러·`hit_test`·`CharClass` 등은 제거됨). `Ctrl+V`/우클릭 붙여넣기는 클립보드 텍스트를 자식 stdin 으로 보내고, `Ctrl+C` 는 인터럽트(`0x03`)를 보낸다(`terminal_keydown`).

#### 스크롤
`update_scrollbar` 가 스크롤백이 있을 때만 스크롤바를 보인다(SetScrollInfo). `OnVScroll`/`OnMouseWheel` 이 `view_top`(통합 인덱스, 총 줄 수 = `scrollback.size() + rows`)을 조정. `print`(새 출력)는 항상 맨 아래로.

### 1.4 확장 지점 / 주의

- **VT 파서 확장**: 새 CSI/ESC 시퀀스는 `dispatch_csi` / `vt_feed` 에 분기를 추가한다. 셀이 `COLORREF` 라 16/256/트루컬러를 모두 표현한다.
- **새 보조 함수**는 파일 내부 전용이면 `ConBox.cpp` 상단의 `static` 함수로 둔다(이식 단위를 `ConBox.h`/`ConBox.cpp` 두 파일로 유지).
- **폰트/창 크기/여백/장평 변경** 시 `calc_cell_size`(폰트만) → `update_metrics`(→ `reset_screen` 으로 화면 재구성) 순으로 다시 계산한다 (`OnSize`, `set_efont/kfont`, `set_margin`, `set_kfont_fill` 참고).
- **장평/높이 보정 비율**은 멤버 `kfill_ratio`(기본 0.92) / `emin_ratio`(기본 0.7)이며 공개 API `set_kfont_fill` 로 바꾼다. `calc_cell_size` 가 한 칸 폭을 정하는 데 쓴다.

#### 현재 미구현/한계 (향후 후보)
- 마우스 드래그 선택/복사·더블클릭 단어 선택 → M3a-2(셀 그리드 좌표로 재작성, 미구현).
- 한글 IME 조합 순서 보장(§3.1)은 코드 구현 완료(`finalize_composition`)이나 실타이핑은 자동 검증 못 해 수동 검증 대상(개발노트 참고). 조합 인라인 직접 렌더는 여전히 시스템 기본에 맡긴다.
- 대체 화면 리사이즈 시 백업(`main_saved`)은 복귀 시점(`leave_alt_screen`→`reset_screen`)에 현재 크기로 맞춘다. 일반 리사이즈 reflow 는 단순(행 수/줄 폭만 맞추고 자르기/패딩).
- UTF-8 멀티바이트가 ConExe 출력 청크 경계에서 잘리면 깨질 수 있다(드묾, 향후 ConExe 측 보완).

## 2. ConExe

ConPTY(의사 콘솔)로 터미널 기반 자식을 실행하고 그 입출력을 ConBox 에 연결하는 헬퍼.
이식 단위는 `Source/ConExe.h` / `ConExe.cpp` 두 파일(클래스 `CConExe`).

### 2.1 구성

- `CConExe` 는 CWnd 파생이 아니다. 출력 폴링 타이머만을 위해 **메시지 전용 창**(`HWND_MESSAGE`)을
  하나 만들어(`timer_wnd`) 거기에 `WM_TIMER` 를 건다. 정적 윈도우 프로시저(`timer_wnd_proc`)가
  `GWLP_USERDATA` 에 보관한 인스턴스로 라우팅해 `pump()` 를 호출한다.
- 주요 멤버: `out_box`(출력 대상), `h_pc`(HPCON), `in_write`(자식 stdin 쓰기), `out_read`(자식
  콘솔 출력 읽기), `proc`(PROCESS_INFORMATION), `timer_wnd`, `running`, `exit_cb`/`exit_cb_user`
  (자식 자연 종료 콜백, `set_exit_callback` 으로 등록).

### 2.2 주요 흐름

#### 실행 `start(cmdline, cols, rows)`
- `CreatePtyPipes`(static): 입력/출력 파이프 쌍 생성 → `CreatePseudoConsole(size, in_read,
  out_write, ...)` 로 HPCON 구성. 우리가 계속 쓰는 끝(`in_write`/`out_read`)은 멤버에 남기고,
  PTY 측 끝(`in_read`/`out_write`)은 호출부로 넘겨 자식 생성 직후 닫는다(닫아야 자식 종료 시
  `out_read` 가 EOF 를 본다).
- `STARTUPINFOEXW` + `PROC_THREAD_ATTRIBUTE_PSEUDOCONSOLE_HANDLE` 속성 목록 구성 →
  `CreateProcessW(EXTENDED_STARTUPINFO_PRESENT)` 로 자식 실행 → 속성 목록/PTY 측 끝 정리 →
  메시지 전용 창 생성 + 타이머(16ms) 시작.

#### 출력 펌프 `pump()` (타이머 폴링)
- `PeekNamedPipe` 로 쌓인 바이트 수를 본 뒤 그만큼만 `ReadFile`(블로킹 회피) → 널 종료해
  `out_box->print()` 로 전달. 받은 UTF-8 은 ConBox 의 `print` 가 VT 시퀀스로 해석한다.
  (후에 워커 스레드로 교체 가능한 지점.)
- **자식 종료 감지**: ConPTY 에서는 자식이 끝나도 conhost 가 출력 파이프의 write 끝을 쥐고 있어
  `PeekNamedPipe`/`ReadFile` 이 EOF 를 못 본다. 그래서 펌프 끝에서 `WaitForSingleObject(proc.hProcess, 0)`
  로 프로세스 종료를 직접 확인하고(남은 출력은 모두 비운 뒤) `handle_child_exit` 로 정리한다.
  `handle_child_exit` 는 `stop()` 으로 정리한 뒤 `exit_cb` 를 호출한다(콜백 시점 `is_running()`==false,
  콜백 안 재시작 안전). Peek 자체 실패(파이프 깨짐)도 같은 경로로 처리한다.

#### 입력·리사이즈 연결 `attach()` / `write()` / `resize()`
- `attach(box)`: `out_box=box` + `box->set_input_sink(&input_thunk, this)`(입력) +
  `box->set_resize_sink(&resize_thunk, this)`(리사이즈)로 양방향+크기를 한 번에 연결한다.
- ConBox 가 키를 인코딩해 `input_thunk` 를 부르면 `write` 가 `in_write` 로 `WriteFile`. 그리드
  크기가 바뀌어 `resize_thunk` 가 불리면 `resize` 가 `ResizePseudoConsole(h_pc, {cols,rows})` 로
  자식 콘솔 크기를 맞춘다. `~CConExe` 는 두 싱크를 먼저 해제(죽은 this 방지) 후 `stop()`.

### 2.3 ConBox 측 터미널(raw) 모드 연동

- 입력 싱크: 멤버 `input_sink`/`input_sink_user`, 공개 `set_input_sink`. 리사이즈 싱크:
  멤버 `resize_sink`/`resize_sink_user`, 공개 `set_resize_sink`. (설정되면 터미널 모드)
- `OnChar`: 터미널 모드면 문자/제어문자를 바이트로 인코딩해 싱크로(헬퍼 `send_input_bytes` /
  `send_input_wide`). 로컬 편집 경로는 없다(터미널 전용화).
- `OnKeyDown`: 터미널 모드면 `terminal_keydown(vk,ctrl,shift)` 가 방향키/편집키를 xterm VT 로
  전송(인쇄 문자/Enter/Backspace/Tab/Esc 는 WM_CHAR 경로에 맡겨 중복 방지).
- IME(`OnImeStart/Comp/End`): 미완성 조합은 시스템 기본 인라인 표시에 맡기고(Default 반환),
  확정분(GCS_RESULTSTR)만 `send_input_wide` 로 보낸 뒤 메시지를 소비(중복 방지).

### 2.4 현재 한계 / 확장 지점

- 자식 콘솔 출력의 VT 시퀀스를 ConBox 가 셀 그리드 위에서 해석한다(커서 이동·ED/EL·SGR
  16/256/트루컬러·autowrap·DECTCEM + 대체 화면·스크롤 영역·줄/문자 삽입삭제·DSR/DA). 창
  리사이즈는 `ResizePseudoConsole` 로 동기화되고, 자식 종료는 `set_exit_callback` 으로 통지된다.
  풀스크린 TUI(vim·claude.exe 등)가 한 경로로 구동된다.
- 미구현/남음: 한글 IME 조합 순서(§3.1)는 코드 구현 완료·실타이핑 수동 검증 대상, 마우스
  선택/복사는 M3a-2. 출력 청크 경계의 UTF-8 멀티바이트 분할 보완은 향후 과제.
