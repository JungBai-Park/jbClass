# 설계명세서

코드를 처음 보는 사람(또는 새 세션)이 resume 없이도 구조를 빠르게 파악하고 손댈 수 있도록 정리한다.
세부 구현은 `Source/` 의 코드가 정본이며, 이 문서는 큰 그림과 확장 지점을 설명한다.

## 1. ConBox

### 1.1 구성

- `Source/ConBox.h` / `ConBox.cpp` — 컨트롤 본체 `CConBox` (CWnd 파생). **이식 단위는 이 두 파일뿐**이다. 파일 내부 전용 보조 함수(`IsWideChar`, 폰트 opts 파서 `ParseFontOpts`, 단어 분류 `CharClass`, 박스/블록 직접 렌더 `DrawBlockElement`/`DrawBoxLine`)는 `ConBox.cpp` 상단의 `static` 함수로 둔다.
- `Project/` — 검증용 MFC 대화상자 데모. `CConBoxDlg` 가 `CConBox` 를 자식으로 띄운다. 컨트롤과 분리되어 있다.

자세한 빌드/관례는 `개발노트.md` 참고.

### 1.2 데이터 모델

```
log_buf : vector<LogLine>           // 논리 줄(Enter 기준) 전체. 출력+입력 내용이 모두 여기 쌓인다.
  LogLine.chars : vector<CharInfo>  // 글자 하나 = CharInfo 하나
    CharInfo { wchar_t ch; COLORREF fg, bg; bool wide; }
      - wide=true : 한글/CJK 등 2배 폭 글자
      - ch==L'\n' : Shift+Enter 로 넣은 "소프트 줄바꿈 마커" (그려지지 않고, 화면 줄만 끊는다)

scr_lines : vector<ScreenLine>      // log_buf 를 가로폭(cols) 으로 자동 줄바꿈한 결과(렌더 캐시)
  ScreenLine { int log_idx, char_start, char_end; }   // log_buf[log_idx].chars[char_start..char_end)
```

상태 변수:
- `cur_log` / `cur_col` — 커서(=출력/입력 위치). log_buf 내 (논리줄, 글자 인덱스).
- `edit_log` / `edit_col` — **편집 경계**. 이 위치 이전은 읽기전용(출력), 이후 ~ 끝이 편집 가능(현재 입력). `print()` 가 끝나면 버퍼 끝으로 전진한다. 보통 마지막 논리 줄.
- `scroll_top` — 화면 맨 위에 보이는 scr_lines 인덱스.
- `cols` / `rows` — 화면에 들어가는 가로 칸 / 세로 줄 수.
- `cell_w` / `cell_h` / `cell_base` — 고정 그리드 한 칸 크기와 베이스라인(픽셀).
- `kfont_match_efont` — 한글을 영문 크기에 맞추는 옵션(기본 true). 별도 토글 API 없이 `set_kfont` 의 크기 인자가 0 이하면 켜지고 양수면 꺼진다. `calc_cell_size` 의 분기를 가른다(아래 그리기/폰트 항목 참고).
- `kfill_ratio` / `emin_ratio` — 맞춤 모드의 한 칸 폭(장평) 비율(기본 0.92 / 0.7). `set_kfont_fill` 로 변경하며 `calc_cell_size` 가 사용한다.
- `cursor_on` / `cursor_blink_ms` — 커서 깜빡임 표시 상태와 토글 간격(ms). `cursor_blink_ms`=0 이면 깜빡이지 않고 항상 표시. 기본 간격은 `GetCaretBlinkTime`.
- `efont_lf` / `kfont_lf` — 사용자가 지정한 폰트 원본 LOGFONT. 장평/높이 보정은 폰트 객체(efont/kfont)에만 적용하고 이 원본은 보존한다(다시 계산 시 자연 폭부터 정확히 측정하기 위함).
- `margin_top` / `margin_bottom` / `margin_left` / `margin_right` — 클라이언트 안쪽 여백(픽셀, 기본 10). `update_metrics` 가 이 여백을 뺀 영역으로 cols/rows 를 계산하고, 모든 픽셀 변환(렌더·커서·hit_test·IME 창)이 좌/상 여백을 더한다. `set_margin` 으로 변경(open 이후면 재계산).
- `desired_vx` — 방향키 위/아래 이동 시 유지할 목표 시각 칸(끈적 열). -1 이면 미설정. 세로 이동 외의 커서 이동/타이핑에서 -1 로 초기화.
- `ime_anchor` — IME 조합 글자가 버퍼에 삽입된 시작 인덱스(`ime_comp` 가 비어있지 않은 동안만 유효).
- `glyph_level` — 박스/블록 그리기 문자 직접 렌더 수준(0 끔 / 1 블록 요소 / 2 +박스선, 기본 1). `set_builtin_glyphs` 로 변경하며 `OnPaint` 의 글자 그리기가 참조한다.
- `back_dc` / `back_bmp` / `back_old_bmp` / `back_w` / `back_h` — 더블 버퍼링용 메모리 DC/비트맵 캐시. `ensure_back_buffer` 가 처음 한 번 만들고 클라이언트 크기가 바뀔 때만 비트맵을 다시 만든다(잦은 다시그리기에서 GDI 객체 재할당 방지).

#### 좌표계 3종 (헷갈리기 쉬움)
- **버퍼 좌표** `(log, col)` — log_buf 내 글자 인덱스. col 은 글자 개수(2배 폭도 1).
- **시각적 칸** `vx` — 화면에서의 가로 칸. 2배 폭 글자는 2칸. (`visual_col()`, 렌더링, 커서 위치 계산에 사용)
- **화면 좌표** `(row, px/py)` — scroll_top 기준 보이는 줄과 픽셀.

### 1.3 주요 흐름

#### 출력 `print(const char* utf8)`
UTF-8 → UTF-16 변환 → 글자별 처리(`\n` 새 논리줄, `\r` 줄 처음, `\t` 4칸 탭스톱, 그 외 `put_wchar`) → `edit_log/edit_col` 을 끝으로 갱신 → `rebuild_scr_lines` → `scroll_to_bottom` → 다시 그림. 색상은 현재 `cur_fg/cur_bg` 를 글자마다 저장.

#### 입력
- `OnChar` — 보이는 글자를 `insert_wchar` 로 커서에 삽입.
- `OnKeyDown` — Back/Delete(편집 경계 안에서만), 방향키 좌/우·Home·End(편집 경계~끝으로 clamp), 방향키 위/아래(`move_cursor_row`), Tab(공백), Enter(`commit_input`), Shift+Enter(소프트 줄바꿈 마커 삽입), Ctrl+Home/End(스크롤만), Ctrl+C/V(복사/붙여넣기).
- `move_cursor_row(dir)` — `cursor_screen_pos` 로 현재 화면 줄·시각 칸을 구해 위/아래 화면 줄(`scr_lines`)로 이동. 같은 논리 줄(`cur_log`) 안에서만, 끈적 열(`desired_vx`)을 목표로 hit_test 와 같은 규칙으로 글자 인덱스를 잡고 줄 끝/편집 경계로 보정. 필요하면 스크롤을 맞춘다.
- `OnGetDlgCode` 이 DLGC_WANTALLKEYS 를 반환해 부모 대화상자가 키를 가로채지 않게 한다.

#### 확정 `commit_input()`
편집 경계~끝의 글자(소프트 줄바꿈은 `\n`)를 모아 UTF-8 로 만들어 **`on_enter` 콜백을 1회** 호출. 새 논리 줄을 추가하고 경계를 전진. (콜백이 `print` 를 다시 불러도 안전하도록 상태 갱신 후 맨 마지막에 호출)

#### 줄바꿈 `rebuild_scr_lines()`
논리 줄마다 폭(cols)을 넘으면 끊고, 소프트 줄바꿈 마커에서 강제로 끊는다. 2배 폭 글자는 줄 끝에서 잘리지 않게 통째로 다음 줄로 넘긴다. 창 크기/버퍼 변경 시 호출.

#### 폰트/셀 크기 `calc_cell_size()`
폰트 메트릭으로 고정 그리드 한 칸 크기 `cell_w`(가로) / `cell_h`(세로) / `cell_base`(베이스라인)를 정한다. DPI 는 `GetDeviceCaps(LOGPIXELSY)` 로 얻어 포인트→픽셀 환산에 쓴다(호스트가 DPI Aware 면 실제 DPI).

**공통 전처리**: 매번 영문을 원본(`efont_lf`)으로 되돌려 만든 뒤 자연 폭부터 측정한다(이전 호출의 장평 잔재 제거).
- `natw` = 영문 `"W"` 폭(`GetTextExtentPoint32`), `eh` = `tmHeight + tmExternalLeading`(영문 줄 높이), `ea` = `tmAscent`.

**`kfont_match_efont` ON (기본, set_kfont 크기 ≤ 0)** — 한글을 영문 높이에 맞추고 칸 폭을 한글 폭에서 역산한다.
1. **한글 높이 맞춤**: 한글을 영문 `lfHeight`(=em) 로 만들어 실제 높이 `kh0` 를 잰다. `kh0 != eh` 면 `newh = lfHeight · eh / kh0` 로 `lfHeight` 를 비례 보정해 다시 만든다(같은 em 만으로는 한글이 더 커 보이므로 실제 글자 높이를 맞춘다). 폭은 자연 폭(`lfWidth=0`) 유지 — 가로로 찌그러뜨리면 획이 깨진다.
2. **한글 폭 측정**: 높이 맞춘 한글 `"가"` 의 자연 폭 `kwid` 를 잰다.
3. **한 칸 폭 `cw` 결정** (장평의 핵심):
   - 1순위 후보: `cw = round(kwid / (2 · kfill_ratio))` — 한글 한 글자가 두 칸을 `kfill_ratio`(기본 0.92)만큼 채우도록 칸 폭을 한글 폭에서 역산.
   - 상·하한으로 자르기: `emin = round(natw · emin_ratio)`(기본 0.7). `cw` 를 `[emin, natw]` 로 clamp(최소 1). 즉 영문을 자연 폭보다 **넓히지 않고**(상한 natw), 가독성 하한(emin) 밑으로 **좁히지도 않는다**. 보통은 후보값이 범위 안이라 하한·상한이 발동하지 않는다.
   - 기본 폰트·DPI 96 실측 예: `natw=9`, `kwid=14` → 후보 `round(14/1.84)=8` → `[6, 9]` 범위 안 → `cw=8`.
4. **영문 장평**: 영문을 `lfWidth = cw` 로 다시 만들어 좁히고, 그 영문 `"W"` 실측 폭이 `cell_w` 가 된다.
5. `cell_h = max(eh, kh)`, `cell_base = max(ea, ka)`(거의 영문과 같다). → 한글은 왜곡 없이 자간만 좁아지고 줄 높이는 영문 기준으로 촘촘해진다.

**`kfont_match_efont` OFF (set_kfont 크기 > 0)**: 영문 원본 폭(`cell_w = natw`), 한글도 `kfont_lf` 원본 크기, `cell_h / cell_base = max(영문, 한글)`.

- 주의: GDI 객체를 DC 에 선택한 채로 `DeleteObject` 하면 안 되므로, 폰트 재생성 전 선택을 해제한다.
- 정수 변환 함정(검증 재현 시): 포인트→픽셀 `-(LONG)(pt·dpi/72 + 0.5)` 와 `cw=(int)(…+0.5)` 는 round-half-up, 높이 비례보정 `(LONG)(…)` 은 0 방향 절사. (개발노트 "폰트 메트릭 검증" 참고)

#### 그리기 `OnPaint()`
**더블 버퍼링**: 메모리 DC(`CreateCompatibleDC` + 호환 비트맵)에 한 프레임을 모두 그린 뒤 마지막에 `BitBlt` 로 화면에 한 번에 복사한다(빠른 입력 시 깜빡임 제거). 이식성을 위해 `CMemDC` 가 아니라 일반 GDI 사용. 모든 픽셀 좌표는 좌/상 여백(`margin_left/top`)을 더한다.
순서: 배경 채움 → 보이는 scr_lines 순회하며 칸 단위로 그림(선택 영역은 fg/bg 반전, 2배 폭은 kfont) → 커서. 베이스라인(TA_BASELINE) 정렬로 한·영 글자 바닥을 맞춘다.
커서(조합 중이 아닐 때)는 칸을 꽉 채우는 **블록**으로, `get_cursor_rect()` 사각형을 `blend_cursor_color()`(배경:전경 = `cursor_bg_weight`:`cursor_fg_weight`, 기본 6:4 혼합, `set_cursor_blend` 로 변경 가능) 으로 채운다. 폭은 한글 모드(`is_hangul_mode()`)면 2칸·영문이면 1칸. 커서 블록이 덮는 글자들은 블록 사각형으로 클리핑해 전경색으로 다시 그려 가려지지 않게 한다(2칸 블록이 다음 칸 글자/한글 왼쪽 절반까지 덮는 경우 포함).
**깜빡임**: `WM_TIMER`(`CURSOR_TIMER`) 의 `OnTimer` 가 `cursor_on` 을 토글하고 커서 칸만 `InvalidateRect`. `OnPaint` 는 `cursor_on && ime_comp.empty()` 일 때만 블록을 그린다(꺼진 프레임엔 본문 글자만 남는다). 간격은 `cursor_blink_ms`(0 이면 타이머 없이 항상 표시). 타이핑/이동/출력 핸들러는 `bump_cursor()` 로 `cursor_on=true` + 타이머 재시작해 입력 직후 한 박자 보이게 한다.
조합 중(`ime_comp` 비어있지 않음)에는 일반 블록 대신, 조합 글자 영역을 **두께 2px 속 빈 사각형**(`blend_cursor_color()` 테두리)으로 감싼다(조합 글자는 본문 글자로 이미 그려져 있고 속은 배경색이 보인다). 이 사각형은 `cursor_on` 과 무관하게 항상 그려 깜빡이지 않는다(`OnTimer` 도 조합 중엔 토글하지 않는다).

#### IME (한글)
`OnImeStart` 조합/후보 창을 커서 위치로. `OnImeComp` 의 GCS_RESULTSTR 은 (먼저 조합 글자 제거 후) `insert_wchar` 로 확정 삽입한다. GCS_COMPSTR(조합 중 문자열)은 **버퍼에 실제 글자로 삽입**한다 — `ime_anchor`(=조합 시작 시 `cur_col`) 위치에 끼워 넣어, 뒤 글자들이 자연히 밀리고 줄바꿈/리플로우가 기존 경로에서 처리된다. 조합 글자 영역은 `OnPaint` 에서 속 빈 사각형으로 표시한다. `clear_ime_comp()` 가 현재 조합 글자를 버퍼에서 제거하고 `cur_col` 을 `ime_anchor` 로 되돌린다(갱신·확정·취소의 공통 전처리). `OnImeEnd` 는 결과 없이 끝난 경우(취소) 조합 글자를 제거해 복구한다. 기본 인라인 그리기는 막고(0 반환) 우리가 그린다.
조합 전 한/영 상태는 `is_hangul_mode()` 가 `ImmGetConversionStatus` 의 `IME_CMODE_NATIVE` 로 조회한다. 이 값으로 커서 블록 폭(한글이면 2칸, 영문이면 1칸)을 정한다. 한/영 토글 시 오는 `OnImeNotify`(WM_IME_NOTIFY / IMN_SETCONVERSIONMODE)가 커서 영역을 무효화해 폭 변화를 즉시 반영한다.

#### 마우스/클립보드
드래그(LButtonDown/Move/Up)로 `(log,col)` 두 점 선택 → 떼면 자동으로 클립보드 복사(CF_UNICODETEXT). 우클릭/Ctrl+V 는 커서에 붙여넣기. `hit_test` 가 픽셀 → `(log,col)` 변환.
`OnLButtonDblClk` 더블클릭 단어 선택: `hit_test` 로 글자를 찾아 `CharClass`(0=구분자/1=한글/2=영문(영숫자)/3=특수문자) 기준으로 같은 부류 연속 구간을 좌우로 넓혀 선택하고 `copy_selection_to_clipboard`. 기존 선택 경로(`get_sel_range`/`pos_selected`/`collect_selection`)를 그대로 재사용한다.

#### 스크롤
`update_scrollbar` 가 내용이 넘칠 때만 스크롤바 표시(SetScrollInfo). `OnVScroll`/`OnMouseWheel` 이 `scroll_top` 조정. `print` 는 항상 맨 아래로.

### 1.4 확장 지점 / 주의

- **색상**: `set_color`/`set_bg_color` 후 `print`/입력에 적용(글자별 저장). VT100(ANSI 이스케이프) 지원을 붙인다면 `print` 의 글자 루프에서 이스케이프 시퀀스를 파싱해 `cur_fg/cur_bg` 를 바꾸는 식으로 확장.
- **새 보조 함수**는 파일 내부 전용이면 `ConBox.cpp` 상단의 `static` 함수로 둔다(이식 단위를 `ConBox.h`/`ConBox.cpp` 두 파일로 유지).
- **폰트/창 크기/여백/장평 변경** 시 `calc_cell_size`(폰트만) → `update_metrics` → `rebuild_scr_lines` 순으로 다시 계산해야 한다 (`OnSize`, `set_efont/kfont`, `set_margin`, `set_kfont_fill` 참고).
- **장평/높이 보정 비율**은 멤버 `kfill_ratio`(기본 0.92) / `emin_ratio`(기본 0.7)이며 공개 API `set_kfont_fill(fill_ratio, emin_ratio)` 로 바꾼다. 한글 자간/영문 폭의 절충값으로 `calc_cell_size` 가 한 칸 폭을 정하는 데 쓴다.

#### 현재 미구현/한계 (향후 후보)
- 가로 스크롤 없음(항상 자동 줄바꿈).
- `\r` 덮어쓰기와 `\t` 는 글자 인덱스 단위로 단순 처리(시각적 칸과 미세하게 다를 수 있음).
- **한글 IME 실시간 조합 과정**은 자동 검증 못 함 → 수동 확인 필요(개발노트 참고).
- 비동기 `print` 가 사용자 입력 도중 끼어들면 입력분이 읽기전용으로 굳는다(엣지 케이스, 명세 밖).
