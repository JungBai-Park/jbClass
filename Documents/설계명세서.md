# ConBox 설계명세서

코드를 처음 보는 사람(또는 새 세션)이 resume 없이도 구조를 빠르게 파악하고 손댈 수 있도록 정리한다.
세부 구현은 `Source/` 의 코드가 정본이며, 이 문서는 큰 그림과 확장 지점을 설명한다.

---

## 1. 구성

- `Source/ConBox.h` / `ConBox.cpp` — 컨트롤 본체 `CConBox` (CWnd 파생). 이식 대상.
- `Source/Accessories.h` / `Accessories.cpp` — 보조 함수. 현재 `ParseFontOpts()` (폰트 opts 문자열 → LOGFONTW).
- `Project/` — 검증용 MFC 대화상자 데모. `CConBoxDlg` 가 `CConBox` 를 자식으로 띄운다. 컨트롤과 분리되어 있다.

자세한 빌드/관례는 `개발노트.md` 참고.

## 2. 데이터 모델

```
log_buf : vector<LogLine>           // 논리 줄(Enter 기준) 전체. 출력+입력 내용이 모두 여기 쌓인다.
  LogLine.chars : vector<CharInfo>  // 글자 하나 = CharInfo 하나
    CharInfo { wchar_t ch; COLORREF fg, bg; bool wide; }
      - wide=true : 한글/CJK 등 2배 폭 글자
      - ch==L'\n' : Shift+Enter 로 넣은 "소프트 줄바꿈 마커" (그려지지 않고, 화면 줄만 끊는다)

scr_lines : vector<ScreenLine>      // log_buf 를 가로폭(cols) 으로 자동 줄바꿈한 결과(렌더 캐시)
  ScreenLine { int log_idx, char_start, char_end; }   // log_buf[log_idx].chars[char_start..char_end)
```

상태 변수:
- `cur_log` / `cur_col` — 커서(=출력/입력 위치). log_buf 내 (논리줄, 글자 인덱스).
- `edit_log` / `edit_col` — **편집 경계**. 이 위치 이전은 읽기전용(출력), 이후 ~ 끝이 편집 가능(현재 입력). `print()` 가 끝나면 버퍼 끝으로 전진한다. 보통 마지막 논리 줄.
- `scroll_top` — 화면 맨 위에 보이는 scr_lines 인덱스.
- `cols` / `rows` — 화면에 들어가는 가로 칸 / 세로 줄 수.
- `cell_w` / `cell_h` / `cell_base` — 고정 그리드 한 칸 크기와 베이스라인(픽셀).

### 좌표계 3종 (헷갈리기 쉬움)
- **버퍼 좌표** `(log, col)` — log_buf 내 글자 인덱스. col 은 글자 개수(2배 폭도 1).
- **시각적 칸** `vx` — 화면에서의 가로 칸. 2배 폭 글자는 2칸. (`visual_col()`, 렌더링, 커서 위치 계산에 사용)
- **화면 좌표** `(row, px/py)` — scroll_top 기준 보이는 줄과 픽셀.

## 3. 주요 흐름

### 출력 `print(const char* utf8)`
UTF-8 → UTF-16 변환 → 글자별 처리(`\n` 새 논리줄, `\r` 줄 처음, `\t` 4칸 탭스톱, 그 외 `put_wchar`) → `edit_log/edit_col` 을 끝으로 갱신 → `rebuild_scr_lines` → `scroll_to_bottom` → 다시 그림. 색상은 현재 `cur_fg/cur_bg` 를 글자마다 저장.

### 입력
- `OnChar` — 보이는 글자를 `insert_wchar` 로 커서에 삽입.
- `OnKeyDown` — Back/Delete(편집 경계 안에서만), 방향키 좌/우·Home·End(편집 경계~끝으로 clamp), Tab(공백), Enter(`commit_input`), Shift+Enter(소프트 줄바꿈 마커 삽입), Ctrl+Home/End(스크롤만), Ctrl+C/V(복사/붙여넣기).
- `OnGetDlgCode` 이 DLGC_WANTALLKEYS 를 반환해 부모 대화상자가 키를 가로채지 않게 한다.

### 확정 `commit_input()`
편집 경계~끝의 글자(소프트 줄바꿈은 `\n`)를 모아 UTF-8 로 만들어 **`on_enter` 콜백을 1회** 호출. 새 논리 줄을 추가하고 경계를 전진. (콜백이 `print` 를 다시 불러도 안전하도록 상태 갱신 후 맨 마지막에 호출)

### 줄바꿈 `rebuild_scr_lines()`
논리 줄마다 폭(cols)을 넘으면 끊고, 소프트 줄바꿈 마커에서 강제로 끊는다. 2배 폭 글자는 줄 끝에서 잘리지 않게 통째로 다음 줄로 넘긴다. 창 크기/버퍼 변경 시 호출.

### 그리기 `OnPaint()`
배경 채움 → 보이는 scr_lines 순회하며 칸 단위로 그림(선택 영역은 fg/bg 반전, 2배 폭은 kfont) → IME 조합 중 문자열(밑줄) → 커서 언더바. 베이스라인(TA_BASELINE) 정렬로 한·영 글자 바닥을 맞춘다.

### IME (한글)
`OnImeStart` 조합/후보 창을 커서 위치로, `OnImeComp` 의 GCS_RESULTSTR 은 즉시 삽입·GCS_COMPSTR 은 `ime_comp` 에 보관해 직접 그림, `OnImeEnd` 에서 비움. 기본 인라인 그리기는 막고(0 반환) 우리가 그린다.

### 마우스/클립보드
드래그(LButtonDown/Move/Up)로 `(log,col)` 두 점 선택 → 떼면 자동으로 클립보드 복사(CF_UNICODETEXT). 우클릭/Ctrl+V 는 커서에 붙여넣기. `hit_test` 가 픽셀 → `(log,col)` 변환.

### 스크롤
`update_scrollbar` 가 내용이 넘칠 때만 스크롤바 표시(SetScrollInfo). `OnVScroll`/`OnMouseWheel` 이 `scroll_top` 조정. `print` 는 항상 맨 아래로.

## 4. 확장 지점 / 주의

- **색상**: `set_color`/`set_bg_color` 후 `print`/입력에 적용(글자별 저장). VT100(ANSI 이스케이프) 지원을 붙인다면 `print` 의 글자 루프에서 이스케이프 시퀀스를 파싱해 `cur_fg/cur_bg` 를 바꾸는 식으로 확장.
- **새 보조 함수**는 `Accessories.*` 에 모은다.
- **폰트/창 크기 변경** 시 `calc_cell_size` → `update_metrics` → `rebuild_scr_lines` 순으로 다시 계산해야 한다 (`OnSize`, `set_efont/kfont` 참고).

### 현재 미구현/한계 (향후 후보)
- 방향키 **위/아래**는 미구현(좌/우·Home·End 만). 입력이 소프트 줄바꿈으로 여러 화면 줄에 걸칠 때 위아래 이동이 필요하면 추가.
- 가로 스크롤 없음(항상 자동 줄바꿈).
- `\r` 덮어쓰기와 `\t` 는 글자 인덱스 단위로 단순 처리(시각적 칸과 미세하게 다를 수 있음).
- **한글 IME 실시간 조합 과정**은 자동 검증 못 함 → 수동 확인 필요(개발노트 참고).
- 비동기 `print` 가 사용자 입력 도중 끼어들면 입력분이 읽기전용으로 굳는다(엣지 케이스, 명세 밖).
