# CLAUDE.md

## 프로젝트 개요
1. 타 프로젝트에 쉽게 이식 가능한 ConBox라는 커스텀 윈도우 컨트롤에 대한 소스코드를 제작한다.
2. 타 프로젝트에 쉽게 이식 가능한 ConExe라는 터미널 기반 응용프로그램 실행기 헬퍼 클래스에 대한 소스코드를 제작한다.



## 요구사항
@Documents/요구사양서.md

"요구사양서 정리해줘"라고 하면 세션 내용에서 내가 추가 요청한 내용들을 반영하여 Documents/요구사양서.md 문서를 최신화 한다.



## 작업 방식 (일반 지침)

@Documents/Karpathy.md

- 답변은 항상 공손하게 "~입니다.",  "~하겠습니다.", "~입니까?" 말투를 사용한다.
- 답변은 항상 한국어로 답한다.
- 작업과정, 추론과정, 생각하는 내용도 가급적 한국말로 표시한다.



## 개발환경

- Language: C++ (MFC, unicode)
- OS : Windows 10 64bit



## 개발 노트

@Documents/개발노트.md

- **[필수]** 이 프로젝트에서 "기억해줘" 등 메모리 저장 요청을 받으면, `~/.claude` 의 메모리 시스템을 절대 사용하지 말고 **반드시** `Documents/개발노트.md` 에 기록한다. 이 규칙은 어떤 경우에도 예외 없이 지킨다.
- 코드 구조(클래스/데이터 모델/주요 흐름/확장 지점)를 파악해야 할 때는 `Documents/설계명세서.md` 를 참고한다.



## 개발 계획

@Documents/개발계획서.md

- ConExe + ConBox 를 풀스크린 터미널 에뮬레이터로 키우는 다단계 작업의 로드맵과 진행상황이다. 여러 세션에 걸쳐 진행되므로, 작업을 이어갈 때는 이 문서의 "진행상황" 체크리스트에서 현재 단계를 확인해 그 지점부터 진행한다.



## 폴더 구조

- 모든 소스코드는 반드시 ./Source 디렉토리 아래에 생성한다.

```plaintext
project-root/
│
├── CLAUDE.md
│
├── Documents/
│    └── 요구사양서.md 등의 설계 참고 문서
│
├── Project/
│    └── Visual Studio의 프로젝트 폴더 : 비주얼 스튜디오가 생성한 파일들
│
├── Source/
│    └── 소스코드 폴더 : 클로드코드가 생성 및 수정 하는 소스코드들은 여기에
│
└── Temp/
     └── 임시 파일 폴더 : 시험/검증용으로 만드는 일회성 파일은 여기에 (gitignore 됨)
```

- **[필수]** 시험·검증을 위해 임시로 만드는 파일(테스트 스크립트, 캡처 이미지, 측정용 ps1, 실험 코드 등)은 프로젝트 루트가 아니라 반드시 `Temp/` 디렉토리 아래에 생성한다. `Temp/` 가 없으면 만들어서 쓴다. (`Temp/` 는 gitignore 되어 있으므로, 작업이 중도 중단되어 임시파일이 남아도 버전관리/루트를 어지럽히지 않는다.)



## 코딩 규칙 (명명·주석)

- 아래 명명/주석 규칙은 Source/ 밑의 소스코드에 적용하며, Project/ 밑의 소스코드에는 억지로 적용하지 않는다.
- 새 모듈 추가 시 `Source/` 아래에 생성할 것 (프로젝트 루트에 .cpp, *.h 파일을 생성하지 말 것)
- 주석은 매우 충실하게 작성
  - 주석에 유니코드 기반 이모티콘 사용 금지
  - 주석은 Doxygen 스타일에 맞출 필요는 없으며 평범한 형태로 작성
  - 함수 내부의 처리과정도 주석을 충실히 적되, 주석안에 1, 2, 3, 4 등의 단계 넘버링은 매기지 말것
- 파일 헤더 코멘트 (나중에 AI 가 코드를 빠르게 파악하고 토큰을 절약하도록)
  - 각 소스 파일 상단에 그 파일의 용도를 설명하는 헤더 코멘트를 둔다.
  - 공개 인터페이스 헤더(.h)에는 용도·의존성/요구사항·최소 사용 예제를 담아, "사용만" 할 때는 그 헤더만 읽으면 충분하도록 자기완결적으로 작성한다.
  - 구현 파일(.cpp)에는 상단에 함수/섹션 목차를 두어, 전체를 통독하지 않고 필요한 부분만 찾아(Grep/부분 읽기) 접근하도록 한다.
  - 코드를 추가·변경할 때 이 헤더 코멘트와 목차도 함께 최신으로 유지한다.
- 파일명/함수/변수 명명 규칙
  - 파일이름 : 

    - PascalCase, 확장자 및 쩜 제외한 이름은 16자 이내 (예: MainCtrl.c)

  - 글로벌 함수 : 

    - PascalCase, 대문자로 시작 (예: FunctionNameSample, MyClassName)
    - 가급적 두 단어 이상으로 조합된 이름을 사용할 것

  - 글로벌 변수 : 

    - camelCase, 소문자로 시작 (예: functionNameSample, myVariableName)
    - 단, 반드시 대문자가 있어야 하기에, 반드시 두 단어 이상으로 조합된 이름을 사용할 것

  - 멤버변수, 멤버 함수, 로컬 변수 : 

    - snake_case, 소문자 + 언더바 (function_name_sample)

    - 단, 로컬변수는 가급적 언더바 없이 짧게 한단어로 표현 (정확한 의미 전달이 필요한 경우는 예외)
    - 단순 루프제어의 iteration 변수는 i, n, k 등의 1개 소문자 변수 사용 
    - 멤버 변수를 뜻하는 접두어(m_)는 불필요
